# ai/logger/

This folder stores inference output logs generated by the model inference scripts.

---

## Folder Structure

| Item | Description |
|---|---|
| `rev0/` | Logs from the rev0 inference runs |
| `llamafactory_infer.txt` | Inference log from LLaMAFactory model runs |
| `log_3epochs.txt` | Inference log from the 3-epoch Pix2Struct model |
| `log_5epochs.txt` | Inference log from the 5-epoch Pix2Struct model |
| `moondream_infer.txt` | Inference log from Moondream model runs |
| `paligemma_infer.txt` | Inference log from PaliGemma model runs |

---

## Log Format

Each log file contains one line per processed image:

```
<image_filename>: <generated_caption>
```

If an image failed during inference, the line will read:

```
<image_filename>: ERROR
<stack_trace>
```

---

## Notes

- Logs are appended to (not overwritten) on each inference run — be mindful of this when re-running scripts
- The 3-epoch vs 5-epoch Pix2Struct logs were used to compare training efficiency; the 3-epoch model was selected as the better trade-off
- LLaMAFactory, Moondream, and PaliGemma logs represent experimental runs — BLIP was selected as the final model
